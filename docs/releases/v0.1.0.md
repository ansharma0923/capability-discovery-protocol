# v0.1.0 Release Notes

**Released**: 2024-01-15

## Overview

Initial release of the Capability Discovery Protocol (CDP). This release establishes the core protocol, reference implementation, and JSON Schema definitions.

## What's New

### Protocol
- DiscoveryIntent schema with 6 categories: product, service, agent, data, compute, api
- ProviderDescriptor and OfferingDescriptor schemas
- 14-stage discovery pipeline
- 4 ranking profiles: default, cost_optimized, latency_optimized, trust_optimized
- Federation support for distributed discovery
- Policy engine with trust-level enforcement

### Reference Implementation (Python)
- FastAPI server with `/discover`, `/register/provider`, `/register/offering`, `/health` endpoints
- In-memory registry with TTL-based expiry
- Keyword-based semantic matching
- Weighted scoring with explainability
- Audit logging

### Schema & Vectors
- 9 JSON Schema Draft 7 files
- 5 protocol vectors for conformance testing

## Known Limitations

- Semantic matching is keyword-based (no vector embeddings in v0.1)
- Registry is in-memory only (no persistence)
- No built-in authentication (must be added in deployment)

## Upgrading

This is the initial release. No migration needed.
